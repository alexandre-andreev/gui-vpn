# Постановка задачи: GUI-утилита управления VPN-подключением (AmneziaWG)

**Версия:** 1.0  **Дата:** 2026-04-30  **Автор:** Александр Андреев

> **Jira:** KAN-2 · Проект: Разработка приложения для управления впн · Статус: Готово

---

## 1. Контекст

На локальной машине (Debian 12) установлен AmneziaWG — модифицированный WireGuard-клиент с обфускацией ([модуль ядра](https://github.com/amnezia-vpn/amneziawg-linux-kernel-module)), используется VPN-провайдер [hidemy.name](https://hide-my-name.biz/faq/vpn/vpn-installation-and-configuration/third-party-applications/wireguard-for-linux/). Скачано ~24 конфигурации серверов разных стран, хранятся в `/home/alexandre/Загрузки/vpn-awg/`. Включение/отключение туннеля сегодня выполняется парой bash-скриптов, которые дёргают `systemctl enable/start/stop/disable awg-quick@wg0.service`.

## 2. Проблема

1. Текущий способ не позволяет переключаться между странами/конфигурациями — нужно вручную подменять `wg0.conf`.
2. Запуск bash-скриптов из консоли неудобен в повседневной работе.
3. Нет визуальной индикации состояния туннеля и доступности интернета.

## 3. Цель

Разработать на Python десктопную GUI-утилиту для Debian 12, которая позволяет в один клик включать, выключать и переключать VPN-подключение AmneziaWG между заранее скачанными конфигурациями серверов, с индикацией состояния соединения. Программа должна **сосуществовать** с уже работающими bash-скриптами и не нарушать их работу.

| ID | Требование |

## 6. Архитектура и компоненты

```
┌─────────────────────────────────────────────────┐
│                    GUI (Tkinter)                │
└──────────────┬──────────────────────────────────┘
               │ events
┌──────────────▼──────────────────────────────────┐
│                  Controller                     │
└──┬───────────┬──────────────┬───────────────────┘
   │           │              │
┌──▼───────┐ ┌─▼─────────┐ ┌──▼────────────────┐
│ Config   │ │ VPN       │ │ Network Checker   │
│ Manager  │ │ Service   │ │                   │
└──────────┘ └───────────┘ └───────────────────┘
```

**Модули:**

- `awg_gui/config_manager.py` — поиск `.conf`, парсинг имени → `Country / City / Server`
- `awg_gui/vpn_service.py` — `pkexec systemctl …`, **атомарная** подмена `wg0.conf`, бэкап
- `awg_gui/network_checker.py` — фоновый поллинг 5–10 секунд
- `awg_gui/autostart.py` — XDG `~/.config/autostart/awg-gui.desktop`
- `awg_gui/controller.py` — orchestration, потоки, rollback
- `awg_gui/ui.py` — Tkinter-окно
- `awg_gui/settings.py` — `~/.config/awg-gui/config.json`
- `awg_gui/logger.py` — логирование
- `awg_gui/__main__.py` — точка входа

**Стратегия совместимости с bash-скриптами (NFR-7, FR-10):**

1. При первом запуске — бэкап `wg0.conf` → `wg0.conf.original`.
2. Подмена через `os.replace(tmp, target)` — атомарно.
3. После операции `wg0.conf` всегда валидный AmneziaWG-конфиг — старые скрипты `systemctl start awg-quick@wg0` работают без изменений.
4. В настройках кнопка «Восстановить исходный wg0.conf» из бэкапа.
5. Аварийное закрытие не оставляет полу-записанный файл благодаря атомарному replace.

## 7. Ключевые сценарии (UC)

- **UC-1. Первый запуск.** Бэкап → выбор → подключение → статус «Подключено: Germany · Berlin».
- **UC-2. Переключение страны.** stop → атомарная подмена → start → проверка интернета.
- **UC-3. Отключение.** `systemctl stop` → статус «Отключено».
- **UC-4. Падение интернета.** 2 цикла без ответа → красный индикатор + сообщение.
- **UC-5. Автозапуск VPN.** `systemctl enable awg-quick@wg0.service`.
- **UC-6. Автозапуск приложения.** Создаётся/удаляется `~/.config/autostart/awg-gui.desktop`.
- **UC-7. Откат к bash-скриптам.** Программа сломана → пользователь запускает старый bash-скрипт → туннель поднимается с последним валидным конфигом.

## 8. Acceptance Criteria

1. На чистой Debian 12 видно ≥20 конфигураций с человекочитаемыми именами.
2. Подключение ≤10 сек от клика до зелёного индикатора.
3. Переключение не оставляет «зависший» туннель — `wg show` показывает только новый интерфейс.
4. Падение сети детектируется в ≤15 сек.
5. Все ошибки `systemctl` отображаются пользователю и пишутся в лог.
6. Приложение корректно завершается без оставленных процессов.
7. **После аварийного закрытия** (kill -9 во время переключения) `wg0.conf` остаётся валидным — bash-скрипт `start awg-quick@wg0` отрабатывает без ошибок.
8. Включение/выключение автозапуска приложения корректно создаёт/удаляет desktop-файл и переживает перезагрузку.
9. Покрытие unit-тестами для `config_manager`, `settings`, парсинга имён ≥80%.

## 12. Риски

| Риск | Митигирование |
| --- | --- |
| Нет прав на `/etc/amnezia/amneziawg/wg0.conf` | PolicyKit-политика + проверка при первом запуске |
| Туннель «висит» при переключении | Таймаут + rollback к предыдущему конфигу |
| `pkexec` запрашивает пароль каждый раз | PolicyKit `auth_admin_keep` (кэш 5 мин) |
| Сбой во время записи `wg0.conf` | Атомарный `os.replace` + бэкап `wg0.conf.original` |
